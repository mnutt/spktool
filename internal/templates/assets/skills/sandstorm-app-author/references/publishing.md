# Publishing

Use this reference for app market and release work:

- app metadata
- screenshots
- changelog and versioning
- signing details
- publication readiness checks

Guidance:

- Separate packaging correctness from app market polish.
- When editing `sandstorm-pkgdef.capnp`, name the exact metadata fields being changed and keep the guidance focused on app-author-facing package metadata.
- When preparing a release, prefer the smallest viable checklist and name the exact files that need edits.
